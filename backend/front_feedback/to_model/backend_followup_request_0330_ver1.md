# 백엔드 후속 요청서 - 모델팀 확인용

## DB와 스토리지 확인

- [ ] backend 저장 결과가 DB에 정상적으로 남는지 확인
- [ ] 실제 이미지 파일이 `mirrai-assets` 스토리지 버킷에도 함께 저장되는지 확인
- [ ] 같은 업로드 건에 대해 DB row와 스토리지 객체를 함께 추적할 수 있는지 확인

설명:

backend는 메타데이터를 DB에 저장하고, 실제 파일은 스토리지에 저장합니다. DB만 보면 전체 저장 여부를 놓칠 수 있으므로, 스토리지 버킷도 함께 확인해야 합니다.

## 캡처 / 분석 / 추천 연결

- [ ] 캡처, 얼굴 분석, 추천, 선택, 상담 기록이 서로 올바르게 연결되는지 확인
- [ ] `storage_snapshot` 정보가 데이터 흐름을 확인하는 데 도움이 되는지 확인

설명:

backend는 파일과 레코드의 흐름을 더 잘 추적할 수 있도록 스냅샷 정보를 추가했습니다. 모델팀은 이 정보를 이용해 어느 단계에서 어떤 데이터가 만들어졌는지 더 쉽게 확인할 수 있습니다.

## 트렌드 시드 동기화

- [ ] 빈 스타일 이름은 동기화에서 제외되는지 확인
- [ ] 같은 스타일 이름이 중복으로 들어오면 하나만 반영되는지 확인
- [ ] `duplicate_count`와 `skipped_count` 값이 납득 가능한지 확인

설명:

이 항목은 트렌드 데이터를 `Style` 테이블에 반영할 때 중복과 빈 값을 줄여서 더 깨끗하게 유지하기 위한 보완입니다.

## 관리자 리포트 스냅샷

- [ ] `report_snapshot`과 `style_report_snapshot`으로 검토가 충분한지 확인
- [ ] 스냅샷 정보만으로도 기본 검토와 디버깅이 가능한지 확인

설명:

backend는 관리자 리포트 확인을 조금 더 쉽게 하도록 스냅샷 정보를 늘렸습니다. 모델팀은 이 정보가 실제 점검에 도움이 되는지만 확인하면 됩니다.

## 워터마크 확인

- [ ] 현재 이미지 기반 워터마크가 검토용 이미지에 적절한지 확인
- [ ] 대각선 반복 워터마크가 보이되 원본 이미지를 너무 가리지 않는지 확인

설명:

backend 워터마크는 이미지 기반으로 바뀌었고, 전면 반복 방식으로 적용되고 있습니다. 모델팀은 최종 검토용 이미지로 보기 적절한지만 확인하면 됩니다.
