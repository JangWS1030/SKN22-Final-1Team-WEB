# 백엔드 수정 요청 리포트 (2026-04-02)

프론트엔드 작업 중 발견된 백엔드 로직의 수정 필요 사항을 정리하여 전달드립니다. 현재 프론트엔드에서의 임시 수정 사항은 모두 원복된 상태이며, 아래 내용을 검토 후 반영 부탁드립니다.

---

## 1. 로그아웃 세션 파기 로직 개선

### 현상 및 문제점
- 현재 `logout_page`에서 `clear_session` 함수들을 통해 특정 키값만 삭제하고 있어, 브라우저의 '뒤로 가기'를 누를 경우 이전 세션이 부분적으로 유지되거나 캐시된 로그인 정보를 통해 재진입이 가능한 현상이 발생함.
- 보안 강화를 위해 서버 측 세션 데이터를 완전히 초기화해야 함.

### 수정 요청 사항
- **대상:** `backend/app/front_views.py` 내 `logout_page` 함수
- **내용:** `request.session.flush()`를 호출하여 서버 측 세션 데이터 삭제 및 세션 쿠키 무효화 처리 요청.

---

## 2. 서버 측 카메라 촬영 이미지 검증 로직 완화

### 현상 및 문제점
- 클라이언트 측(MediaPipe)에서는 정밀하게 얼굴 검출이 완료되어 촬영이 진행되었으나, 서버 측(OpenCV Cascade) 검증 단계에서 얼굴을 찾지 못해(`no_face_detected`) `NEEDS_RETAKE`를 반환하는 불일치 현상이 잦음.
- 이로 인해 사용자가 촬영 후 다음 단계(설문)로 넘어가지 못하고 카메라 페이지에 갇히는 현상 발생.

### 수정 요청 사항
- **대상:** `backend/app/services/capture_validation.py` 내 `validate_capture_image` 함수 및 관련 상수
- **제안 사항:**
  - `MIN_FACE_SIZE_RATIO`: 0.08 -> 0.05 로 하향.
  - `MIN_SHARPNESS`: 45.0 -> 35.0 로 하향.
  - `face_count == 0`일 때 무조건 실패 대신, 클라이언트 검증을 신뢰하여 폴백(Fallback) 처리 검토.

## 3. 파트너 센터 페이지별 세션 권한 및 경로 확정

### 현상 및 요구사항
- 파트너 센터 내에서 '매장 관리자'와 '개별 디자이너'의 역할과 접근 권한을 분리하여 운영하고자 함.

### 수정 요청 사항
- **대상:** `backend/app/urls_front.py` 및 세션 체크 로직
- **페이지별 정의 및 경로 변경:**
  1. **`/partner/dashboard/`** (기존 `/partner/staff/`에서 명칭 변경 요청): 
     - **권한:** 매장 사업자 로그인(`admin_id`) 세션 필수.
     - **용도:** 매장 전체 관리, 소속 디자이너 목록 관리, 전체 트렌드 확인.
  2. **`/partner/staff/`**: 
     - **권한:** 디자이너 PIN 인증(`designer_id`) 세션 필수.
     - **용도:** 해당 디자이너 전담 고객 관리 및 상담 관리.

## 4. 고객 리스트 API 데이터 필드 확장 및 변경 요청

### 현상 및 요구사항
- 대시보드 내 고객 목록의 항목이 `고객명 / 디자이너 / 최근 방문 / 총 방문수`로 확정됨에 따라 API 응답 필드 수정이 필요함.

### 수정 요청 사항
- **대상:** `api/v1/customers/` (또는 관련 고객 목록 반환 API)
- **추가/변경 요청 필드:**
  1. `designer_name`: 해당 고객 담당 디자이너 성함. (디자이너 목록 API의 이름과 일치 필수)
  2. `last_visit_date`: 고객의 마지막 방문일 또는 마지막 분석 생성 일시.
  3. `visit_count`: 해당 고객의 총 상담/분석 횟수 (정수형).

## 8. 고객 목록 조회 성능 최적화 요청

### 현상 및 문제점
- 고객 데이터가 100개 이상으로 늘어남에 따라 대시보드 로딩 속도가 현저히 느려짐 (N+1 쿼리 발생 추정).

### 수정 요청 사항
- **쿼리 최적화:** 
  - Django의 `select_related('designer')` 및 `prefetch_related`를 사용한 쿼리 통합.
  - `visit_count` 계산 시 DB 수준의 `annotate(Count(...))` 활용 권장.
- **DB 인덱스:** 
  - `name`, `shop_id`, `designer_id`, `created_at` 필드에 DB 인덱스 설정 검토.
